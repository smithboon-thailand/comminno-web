# คู่มือการจัดการเนื้อหาเว็บไซต์ Comm.Inno (Editorial Handbook)

คู่มือนี้อธิบายขั้นตอนการทำงาน (Workflow) สำหรับทีมบรรณาธิการในการอัปเดตเนื้อหาเว็บไซต์ Comm.Inno ผ่านระบบจัดการเนื้อหา (CMS)

---

## 1. ภาพรวมของระบบ (Architecture Overview)

เว็บไซต์ Comm.Inno ทำงานแบบ **Static Site** (ไม่มีฐานข้อมูลหลังบ้าน) เนื้อหาทั้งหมดถูกเก็บเป็นไฟล์ข้อความ (YAML และ Markdown) ในโฟลเดอร์ `content/` บน GitHub

เราใช้ **Sveltia CMS** เป็นหน้ากาก (Interface) ให้ทีมบรรณาธิการทำงานได้ง่ายขึ้น โดยไม่ต้องเขียนโค้ด:
1. คุณล็อกอินเข้า Sveltia CMS ผ่านเบราว์เซอร์
2. เมื่อคุณกด "Save" ระบบจะสร้าง **Pull Request (PR)** บน GitHub อัตโนมัติ
3. ผู้ดูแลระบบ (Reviewer) ตรวจสอบและกด "Merge"
4. Vercel จะนำเนื้อหาใหม่ไปสร้างเว็บไซต์และอัปเดตให้คนทั่วไปเห็น (Auto-deploy) ภายใน 1-2 นาที

---

## 2. การเข้าสู่ระบบ (Login & Access)

1. ไปที่ **[comminno-web.vercel.app/admin/](https://comminno-web.vercel.app/admin/)**
2. คลิกปุ่ม **Login with GitHub**
3. หากคุณยังไม่มีสิทธิ์เข้าถึง ระบบจะแจ้งเตือน (ดูหัวข้อ "การขอสิทธิ์" ด้านล่าง)

### การขอสิทธิ์เข้าถึง (Collaborator Invite Procedure)
เนื่องจากเนื้อหาผูกกับ GitHub Repository โดยตรง ผู้เขียนทุกคนต้องมีบัญชี GitHub และได้รับเชิญเป็น Collaborator:
1. สมัครบัญชีที่ [github.com](https://github.com) (ฟรี)
2. ส่ง **GitHub Username** ของคุณให้ผู้ดูแลระบบ (Admin)
3. ผู้ดูแลระบบจะส่งคำเชิญ (Invite) ไปยังอีเมลของคุณ
4. **สำคัญ:** คุณต้องกดรับคำเชิญ (Accept Invitation) ในอีเมล หรือไปที่หน้าระบบ GitHub เพื่อกดยอมรับก่อน จึงจะล็อกอินเข้า CMS ได้

## 3. โครงสร้างเนื้อหา (Content Collections)

ในแถบเมนูด้านซ้ายของ CMS คุณจะพบหมวดหมู่เนื้อหาดังนี้:

- **Insights (บทความ):** ข้อมูลเบื้องต้นของบทความ (ชื่อเรื่อง, วันที่, รูปปก, หมวดหมู่)
- **Insight bodies (เนื้อหาบทความ):** เนื้อหาฉบับเต็มของบทความ (เขียนด้วย Markdown) *หมายเหตุ: ต้องตั้งชื่อ Slug ให้ตรงกับ Insights*
- **Services (บริการ):** แคตตาล็อกบริการของศูนย์ฯ
- **About page (หน้าแนะนำศูนย์ฯ):** ข้อมูลพันธกิจ, ผู้บริหาร, ทีมงาน, และพันธมิตร
- **Categories (หมวดหมู่):** ป้ายกำกับ (Tags) สำหรับจัดกลุ่มบทความ
- **Redirects (เปลี่ยนเส้นทาง):** สำหรับตั้งค่า URL เก่าให้วิ่งมาหน้าใหม่ (มักใช้ตอนย้ายเว็บจาก Wix)

*ดูรายละเอียดฟิลด์ทั้งหมดได้ที่ไฟล์ `content/SCHEMA.md`*

## 4. การจัดการรูปภาพ (Image Upload Workflow)

เว็บไซต์ Comm.Inno ใช้ **Cloudinary** เป็นระบบจัดการรูปภาพหลัก เพื่อให้เว็บโหลดเร็วและรองรับภาพสมัยใหม่ (WebP/AVIF) อัตโนมัติ

**ขั้นตอนการอัปโหลดรูปภาพใหม่:**
1. ไปที่ [Cloudinary Console](https://cloudinary.com/) และล็อกอินด้วยบัญชีของศูนย์ฯ
2. ไปที่โฟลเดอร์ที่เกี่ยวข้อง (เช่น `comminno/insights/` หรือ `comminno/team/`)
3. อัปโหลดรูปภาพของคุณ (แนะนำให้ตั้งชื่อไฟล์เป็นภาษาอังกฤษ ตัวพิมพ์เล็ก และใช้ขีดกลางแทนช่องว่าง เช่น `chula-zero-waste-event.jpg`)
4. คัดลอก **Public ID** ของรูปภาพ (เช่น `comminno/insights/chula-zero-waste-event`)
5. นำ Public ID ไปวางในช่อง `coverImage`, `heroImage`, หรือ `photo` ใน Sveltia CMS

*หมายเหตุ: Sveltia CMS มีปุ่ม Media Library แต่สำหรับโปรเจกต์นี้ เราบังคับใช้ Cloudinary URL/Public ID เพื่อประสิทธิภาพสูงสุด กรุณาอัปโหลดผ่าน Cloudinary โดยตรง*

## 5. ขั้นตอนการเขียนและเผยแพร่บทความ (Publishing Workflow)

1. **สร้างข้อมูลเบื้องต้น:** ไปที่หมวด **Insights** กดปุ่ม `New` กรอกข้อมูลให้ครบถ้วน (Slug, Title, Summary, Cover Image) แล้วกด `Save`
2. **สร้างเนื้อหา:** ไปที่หมวด **Insight bodies** กดปุ่ม `New` 
   - **สำคัญมาก:** ช่อง `Slug` ต้องกรอกให้ **ตรงกันเป๊ะ** กับ Slug ที่ตั้งไว้ในข้อ 1
   - เขียนเนื้อหาในช่อง Body (รองรับการจัดรูปแบบด้วย Markdown)
   - กด `Save`
3. **รอการอนุมัติ:** การกด Save จะสร้าง Pull Request บน GitHub
4. **ตรวจสอบและเผยแพร่:** ผู้ดูแลระบบเข้าไปที่ GitHub ตรวจสอบความถูกต้อง และกด Merge PR
5. **ออนไลน์:** รอประมาณ 1-2 นาที เว็บไซต์จะอัปเดตอัตโนมัติ

## 6. ข้อควรระวังและแนวทางปฏิบัติ (Best Practices)

- **Bilingual First (ภาษาไทยเป็นหลัก):** เว็บไซต์นี้ออกแบบมาเพื่อรองรับ 2 ภาษา โดยมีภาษาไทยเป็นภาษาหลัก ฟิลด์ภาษาไทย (เช่น `titleTh`, `summary`) บังคับให้กรอกเสมอ ส่วนภาษาอังกฤษหากไม่มีให้เว้นว่างไว้ (ระบบจะดึงภาษาไทยไปแสดงแทน)
- **Slug ห้ามเปลี่ยน:** `Slug` คือส่วนหนึ่งของ URL (เช่น `comminno.center/th/insights/chula-zero-waste`) หากบทความเผยแพร่ไปแล้ว **ห้ามเปลี่ยน Slug เด็ดขาด** เพราะจะทำให้ลิงก์ที่แชร์ไปแล้วเสีย (404 Not Found)
- **การเว้นวรรคและย่อหน้า:** ในช่องที่เป็นข้อความยาว (เช่น `bioTh`, `descriptionTh`) การกด Enter 1 ครั้งคือการขึ้นบรรทัดใหม่ การกด Enter 2 ครั้งคือการขึ้นย่อหน้าใหม่
- **Alt Text สำคัญเสมอ:** รูปภาพทุกรูปต้องมีคำอธิบาย (Alt Text) เพื่อให้ผู้พิการทางสายตาสามารถเข้าถึงเนื้อหาได้ และเป็นผลดีต่อ SEO
